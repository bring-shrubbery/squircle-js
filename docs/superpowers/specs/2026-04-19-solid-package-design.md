# `@squircle-js/solid` — Solid.js port of the Squircle component

## Summary

Add a new publishable package `@squircle-js/solid` that provides the same squircle-clipping API as `@squircle-js/react`, implemented idiomatically in Solid.js. This is the first step of a broader multi-framework initiative (Vue / Svelte to follow in later specs). The package lives in the same monorepo, is built by the existing `turbo run build`, and is auto-published by the existing Changesets-based release workflow.

## Goals

- Ship `@squircle-js/solid` to npm with API parity to `@squircle-js/react`.
- Keep the new package idiomatic for Solid users (primitives, `children()`, `spread`, `onMount`).
- Require zero manual release-engineering work per package beyond writing changesets.
- Add a Solid usage section to the existing docs site.
- Add a minimal smoke test to catch gross regressions.

## Non-goals

- Vue or Svelte ports (separate future specs).
- Live Solid demos embedded in the React-based docs site — code snippets only.
- Extensive test coverage — parity with the (currently untested) React package plus one smoke test.
- Renaming the existing React package folder (`packages/squircle-element-react` → `packages/react`) — explicitly deferred.

## Component API

The public API mirrors `@squircle-js/react` 1:1, translated to Solid:

### `<Squircle>`

Dynamic, size-observing component. Re-computes the clip path whenever its measured size or corner props change.

Props (all optional unless noted):

- `cornerRadius?: number`
- `cornerSmoothing?: number` — defaults to `0.6`
- `width?: number` — overrides measured width
- `height?: number` — overrides measured height
- `defaultWidth?: number` — used on SSR / before first measurement
- `defaultHeight?: number` — used on SSR / before first measurement
- `asChild?: boolean` — if true, applies styles to the child element via `spread`; otherwise renders a `<div>`
- `children?: JSX.Element`
- plus any `JSX.HTMLAttributes<HTMLDivElement>` — forwarded to the rendered element

Behavior:

- If `asChild` is false (default), renders `<div>` with the computed styles.
- If `asChild` is true and `children()` resolves to a single `Element`, the styles and `data-squircle` attribute are applied to that child via `spread`.
- If `asChild` is true but the resolved child is not a single `Element`, falls back to rendering a `<div>` wrapping the children (matching the safest interpretation — no silent failure).
- The computed style mirrors the React package: `borderRadius: cornerRadius`, `width`/`height` only set when explicitly provided or defaulted, and `clip-path: path('<svg path>')`.
- Path is computed with `createMemo`; returns `""` (no clip-path applied) when measured size is `0×0`.

### `<StaticSquircle>`

For known dimensions — no observer, no memo of a size signal. Takes `width`, `height`, `cornerRadius`, `cornerSmoothing` as required props (matching the React API), optional `asChild`, and forwards remaining HTML attrs. Computes the clip path with `createMemo` over its four inputs.

### `<SquircleNoScript>`

Renders `<noscript><style>[data-squircle] { clip-path: none !important; border-radius: attr(data-squircle) !important; }</style></noscript>`. No dynamic logic; trivial port.

## Key implementation details

### `asChild` / polymorphic helper

Solid has no Radix-equivalent `<Slot>`. The new `polymorphic.tsx` exports a small helper (approximate shape):

```tsx
import { children, type JSX } from "solid-js";
import { spread } from "solid-js/web";

export function renderAsChild(
  getChildren: () => JSX.Element,
  props: Record<string, unknown>
): JSX.Element {
  const resolved = children(getChildren);
  const node = resolved();
  if (node instanceof Element) {
    spread(node, props, false, true);
    return node;
  }
  return null; // caller decides fallback
}
```

The `Squircle` component decides, per render, whether to call this helper (when `asChild` is true and the child resolves to a single `Element`) or to render its own `<div>`. When `asChild` is true but the child doesn't resolve to an element, `Squircle` falls back to a wrapping `<div>` rather than silently dropping content.

### Element size measurement

Uses `createElementSize` from `@solid-primitives/resize-observer`. The primitive returns reactive `width` / `height` signals tied to a `ref`. The measurement only fires on the client (the underlying `ResizeObserver` is set up in effect/`onMount`), so SSR-safety is free.

On the server, `width` and `height` read as `null` / initial defaults; `Squircle` falls back to `defaultWidth` / `defaultHeight` (or renders no clip-path when neither is provided). After hydration, `onMount` fires, the observer attaches, and the real size flows through.

### Path generation

```tsx
const path = createMemo(() => {
  const w = props.width ?? size.width ?? props.defaultWidth ?? 0;
  const h = props.height ?? size.height ?? props.defaultHeight ?? 0;
  if (w === 0 || h === 0) return "";
  return getSvgPath({
    width: w,
    height: h,
    cornerRadius: props.cornerRadius,
    cornerSmoothing: props.cornerSmoothing ?? 0.6,
  });
});
```

## Testing

One Vitest smoke test at `packages/solid/src/Squircle.test.tsx`:

- Uses `@solidjs/testing-library` + `jsdom`.
- Renders `<Squircle cornerRadius={16} defaultWidth={100} defaultHeight={100}>content</Squircle>`.
- Asserts the rendered element has `data-squircle="16"` and a `clip-path` style value starting with `path(` (using `defaultWidth`/`defaultHeight` so the assertion doesn't depend on a mocked `ResizeObserver` firing).
- Needs `vite-plugin-solid` in a `vitest.config.ts` at the package level and a jsdom-based test environment.

The test is a smoke test — its job is to catch catastrophic regressions (broken bundle, broken JSX transform, missing reactive wiring), not to assert pixel-accurate behavior.

No tests added to the React package as part of this spec.

## Release flow

1. Developer makes a change under `packages/solid/`.
2. Developer runs `pnpm changeset` and picks `@squircle-js/solid` + bump type.
3. On merge to `main`, the existing `release.yml` Create Release Pull Request job opens (or updates) a version-bump PR.
4. When that PR merges, the release job runs `pnpm run changeset:release` which builds all packages and publishes those with pending version bumps — Solid included, automatically.

## Out of scope / explicitly deferred

- Vue and Svelte packages.
- Renaming `packages/squircle-element-react` → `packages/react`.

- Cross-package shared internals (e.g., a `@squircle-js/core` with framework-agnostic helpers). If we find ourselves duplicating meaningful logic when we add Vue/Svelte, that's the point to design a core package — not now.
