# `@squircle-js/svelte` — Svelte 5 port of the Squircle component

## Summary

Add a new publishable package `@squircle-js/svelte` that provides the same squircle-clipping API as `@squircle-js/react` and `@squircle-js/solid`, implemented idiomatically in Svelte 5. This is the third framework port following the Solid spec (2026-04-19). The package lives in the same monorepo, is built by the existing `turbo run build`, and is auto-published by the existing Changesets-based release workflow.

## Goals

- Ship `@squircle-js/svelte` to npm with API parity to `@squircle-js/react` and `@squircle-js/solid`.
- Keep the new package idiomatic for Svelte 5 users (runes, actions, `$props()`, `$effect()`).
- Provide the idiomatic Svelte equivalent of `asChild` via an **action** (`use:squircle`), since snippets don't expose a DOM element reference the way React children or Solid's `children()` do.
- Require zero manual release-engineering work per package beyond writing changesets.
- Add Svelte-grouped documentation pages to the existing docs site matching React/Solid coverage depth.
- Add a minimal smoke test.

## Non-goals

- Svelte 4 support — Svelte 5 is the baseline. Peer dep: `svelte: ^5.0.0`.
- Vue port (separate future spec).
- Live Svelte demos embedded in the React-based docs site — code snippets only.
- Extensive test coverage — parity with the React/Solid packages (one smoke test).
- A shared `@squircle-js/core` package — still deferred until pattern becomes painful.

## Component & Action API

Public API mirrors React/Solid 1:1 where idiomatic, with one intentional deviation: **`asChild` is replaced by actions**.

### `<Squircle>` component

Dynamic, size-observing component. Re-computes clip-path whenever measured size or corner props change.

Props (all optional unless noted):

- `cornerRadius?: number`
- `cornerSmoothing?: number` — defaults to `0.6`
- `width?: number` — overrides measured width
- `height?: number` — overrides measured height
- `defaultWidth?: number` — used on SSR / before first measurement
- `defaultHeight?: number` — used on SSR / before first measurement
- `children?: Snippet` — standard Svelte 5 children snippet
- `class?: string` — forwarded to the rendered `<div>`
- `style?: string` — forwarded (additional styles; squircle's clip-path is appended)
- plus any `HTMLAttributes<HTMLDivElement>` via `$$restProps` / rune-style rest

Behavior:

- Renders `<div>` with computed `clip-path`, `border-radius`, optional `width`/`height`, and `data-squircle` attribute.
- Uses the internal `squircle` action (see below) under the hood — the component is essentially a thin `<div use:squircle={opts}>{@render children?.()}</div>` wrapper.
- SSR-safe: when `defaultWidth`/`defaultHeight` are provided, initial HTML includes a computed `clip-path`. When not, initial render has no clip-path until the action mounts on the client.

### `<StaticSquircle>` component

For known dimensions. Takes `width`, `height`, `cornerRadius`, `cornerSmoothing` as required props, plus optional `class`, `style`, `children`. Computes the path synchronously and applies it via inline style — no ResizeObserver. Used internally via the `staticSquircle` action.

### `<SquircleNoScript>` component

Renders:

```html
<svelte:head>
  <noscript>
    <style>
      [data-squircle] { clip-path: none !important; border-radius: attr(data-squircle) !important; }
    </style>
  </noscript>
</svelte:head>
```

Trivial port. Uses `<svelte:head>` so the `<noscript>` lands in `<head>` where it belongs.

### `use:squircle` action

The idiomatic Svelte replacement for `asChild`. Attaches squircle clipping directly to any element without a wrapper.

```ts
import type { Action } from "svelte/action";

export interface SquircleActionOptions {
  cornerRadius?: number;
  cornerSmoothing?: number;
  width?: number;
  height?: number;
  defaultWidth?: number;
  defaultHeight?: number;
}

export const squircle: Action<HTMLElement, SquircleActionOptions>;
```

Behavior:

- Sets `data-squircle={cornerRadius}` on the element.
- Sets `element.style.borderRadius`, `element.style.clipPath`, optional explicit `width`/`height`.
- Observes element size via `ResizeObserver` (unless explicit `width`/`height` given).
- Recomputes path on size change or options update.
- Cleans up observer on destroy.
- Preserves any `style` already present on the element (only sets specific properties, doesn't replace the whole attribute).

Usage:

```svelte
<button use:squircle={{ cornerRadius: 12, cornerSmoothing: 0.6 }}>
  Click me
</button>
```

### `use:staticSquircle` action

Synchronous variant — no observer. Required options: `width`, `height`, `cornerRadius`, `cornerSmoothing`.

```svelte
<img
  src="/avatar.jpg"
  alt="Avatar"
  use:staticSquircle={{ width: 48, height: 48, cornerRadius: 12, cornerSmoothing: 0.6 }}
/>
```

## Key implementation details

### Path generation (shared)

`computePath.ts` exports a single function:

```ts
import { getSvgPath } from "figma-squircle";

export function computeClipPath(opts: {
  width: number;
  height: number;
  cornerRadius?: number;
  cornerSmoothing?: number;
}): string {
  if (opts.width === 0 || opts.height === 0) return "";
  return getSvgPath({
    width: opts.width,
    height: opts.height,
    cornerRadius: opts.cornerRadius,
    cornerSmoothing: opts.cornerSmoothing ?? 0.6,
  });
}
```

Used by both components and both actions.

### `squircle` action — size observation

```ts
export const squircle: Action<HTMLElement, SquircleActionOptions> = (node, options = {}) => {
  let opts = options;
  let observer: ResizeObserver | null = null;

  const apply = (width: number, height: number) => {
    const path = computeClipPath({ width, height, ...opts });
    node.style.clipPath = path ? `path('${path}')` : "";
    if (opts.cornerRadius !== undefined) {
      node.style.borderRadius = `${opts.cornerRadius}px`;
      node.setAttribute("data-squircle", String(opts.cornerRadius));
    }
    if (opts.width !== undefined) node.style.width = `${opts.width}px`;
    if (opts.height !== undefined) node.style.height = `${opts.height}px`;
  };

  const measure = () => {
    const w = opts.width ?? node.offsetWidth ?? opts.defaultWidth ?? 0;
    const h = opts.height ?? node.offsetHeight ?? opts.defaultHeight ?? 0;
    apply(w, h);
  };

  if (opts.width !== undefined && opts.height !== undefined) {
    apply(opts.width, opts.height);
  } else {
    measure();
    observer = new ResizeObserver(measure);
    observer.observe(node);
  }

  return {
    update(newOptions: SquircleActionOptions) {
      opts = newOptions ?? {};
      measure();
    },
    destroy() {
      observer?.disconnect();
    },
  };
};
```

### Component using the action

```svelte
<!-- Squircle.svelte -->
<script lang="ts">
  import type { Snippet } from "svelte";
  import type { HTMLAttributes } from "svelte/elements";
  import { squircle } from "./squircle.svelte.js";

  interface Props extends HTMLAttributes<HTMLDivElement> {
    cornerRadius?: number;
    cornerSmoothing?: number;
    width?: number;
    height?: number;
    defaultWidth?: number;
    defaultHeight?: number;
    children?: Snippet;
  }

  let {
    cornerRadius,
    cornerSmoothing = 0.6,
    width,
    height,
    defaultWidth,
    defaultHeight,
    children,
    ...rest
  }: Props = $props();
</script>

<div
  {...rest}
  use:squircle={{ cornerRadius, cornerSmoothing, width, height, defaultWidth, defaultHeight }}
>
  {@render children?.()}
</div>
```

Identical skeleton for `StaticSquircle.svelte`, using `staticSquircle` action.
